これは質問です。実行に成功しましたが、Request&Responseのターンアラウンドで300～400msくらいかかります。
これはUSまでの距離とe2-microのインスタンスの非力さによるものと考えられますか
はい、その通りです。主に**距離（ネットワークレイテンシ）**が支配的な要因です。

内訳の目安：

要因	推定
日本 → us-central1（アイオワ）RTT	150〜200ms
Request + Response で 2往復（送信＋応答返送）	× 2 = 300〜400ms
e2-micro の処理遅延	数ms（ほぼ無視できる）
gRPC の Request が日本→US に飛び、Broker が相手に中継し、相手の Response がまた日本→US→日本と返ってくるので、太平洋横断が最低2回発生します。これが 300〜400ms の主要因です。

e2-micro は共有 vCPU でバースト制限がありますが、メッセージの JSON パース＆中継程度の処理は数ms で終わるので、インスタンスの非力さはほとんど影響していません。

もしレイテンシを改善したい場合は、asia-northeast1（東京）にリージョンを変更すれば RTT が 10〜20ms 程度になり、50ms 以下まで改善できます。ただし東京リージョンは Always Free 枠の対象外なので課金が発生します。
